## 📌 Why This Framework?
In an era of increasing software supply chain attacks and "protestware," choosing a "free" library is a high-stakes business decision. Integrating a volatile project leads to:
* **The Migration Trap:** Massive costs when a project is abandoned.
* **Security Debt:** Slow patching cycles in solo-maintained tools.
* **Compliance Failure:** Lack of transparency in governance and licensing.

This framework provides a structured way to separate **Hobby Projects** from **Enterprise-Ready Software.**

## 🛠️ The 3 Pillars of Evaluation
This light version introduces the core pillars used in professional IT governance:

1. **Contributor Dedication:** Assessing the "Bus Factor" and corporate backing.
2. **Community Activity:** Measuring the health of the support ecosystem.
3. **End-User Profile:** Verifying production-grade adoption across industries.
