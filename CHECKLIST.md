# OSS Enterprise Readiness Checklist

Use this checklist for an initial "vibe check" of any open-source project. 

### Pillar 1: Contributor Dedication
- [ ] **LTS Version:** Is there a designated Long-Term Support version or a clear release roadmap?
- [ ] **Corporate Backing:** Is the project maintained by a funded company or a neutral foundation (Apache, CNCF)?
- [ ] **Commit Frequency:** Has there been code activity in the last 90 days? (Last update >6 months is a Red Flag).

### Pillar 2: Community Activity
- [ ] **Documentation:** Is the documentation versioned and comprehensive, or just a README?
- [ ] **Issue Response:** Check the 'Issues' tab. Are critical bugs acknowledged within 7–14 days?
- [ ] **Ecosystem Pulse:** Is there high activity on official forums, Slack, or StackOverflow?

### Pillar 3: End-User Profile
- [ ] **Production Proof:** Can you find 3+ reputable companies using this in production? (Check `ADOPTERS.md`).
- [ ] **Industry Diversity:** Is the software used across different sectors, or is it niche?
- [ ] **Industry Standards:** Does the project have an OpenSSF "Passing" or "Silver" badge?
