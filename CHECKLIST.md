# OSS Enterprise Readiness Checklist

Use this reduced checklist for an initial "vibe check" of any open-source project. 

### Pillar 1: Contributor Dedication
- [ ] **LTS Version:** Is there a designated Long-Term Support version or a clear release roadmap?


### Pillar 2: Community Activity
- [ ] **Documentation:** Is the documentation versioned and comprehensive, or just a README?


### Pillar 3: End-User Profile
- [ ] **Production Proof:** Can you find 3+ reputable companies using this in production? (Check `ADOPTERS.md`).
